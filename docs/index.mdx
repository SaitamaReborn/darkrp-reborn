## Créez des addons — et vendez-les si vous voulez

Le système d'addons est pensé pour que **n'importe qui puisse étendre le jeu sans jamais accéder à notre code source ni à nos bases de données** : vous codez contre un contrat d'API public, isolé par design. Votre addon reste le vôtre — crédité en jeu, et vendable si c'est votre modèle.

Votre premier addon C#, en cinq minutes :

```csharp
public sealed class MonAddon : RebornAddon
{
	public override string Ident => "mon-addon";
	public override string Name => "Mon premier addon";

	public override void OnAddonLoaded()
	{
		RebornEvents.PlayerJoined += p => p.Chat( $"Bonjour {p.Name} !" );
		Reborn.RegisterChatCommand( "salut", "Dit bonjour", ( player, args ) =>
			player.Notify( "Salut !", $"Tu as {player.Money}$ en poche." ), Ident );
	}
}
```

Aucun enregistrement : chaque classe dérivée de `RebornAddon` est découverte et chargée automatiquement, côté serveur. Le même code tourne dans le simulateur du Dev Kit et en production. [Démarrer maintenant →](/addons/csharp-demarrage)

## L'esprit du projet

- **Vos addons, votre serveur.** Aucune règle de gameplay imposée : créez les métiers et systèmes que VOUS voulez, gardez la propriété de votre code.
- **Sécurité d'abord.** Chaque addon tourne isolé : pas d'accès aux bases de données, persistance sandboxée, budget CPU par appel côté Lua, économie serveur-autoritaire côté C#.
- **Compatibilité mesurée, pas promise.** Chaque API GMod est validée sur de vrais addons publics — [la matrice complète est publique](/compat/darkrp).
