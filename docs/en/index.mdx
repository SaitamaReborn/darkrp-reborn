## Two ways to build addons — and even sell them

- **Lua (GMod compatible)**: folders of scripts dropped on your server, hot-reloadable - ideal for migrating what you have and scripting fast. This is the SDK's "coming from Garry's Mod" half.
- **C# (native s&box)**: the [Dev Kit](/en/addons/csharp-getting-started) simulates a DarkRP Reborn server in the editor; you code against the public API contract (`Reborn`, `RebornEvents`, `IRebornPlayer`) and the addon is integrated into the game after review. Both worlds see the same game events.

## How it works

Your GMod Lua scripts run as-is, GLua syntax included, with the API you already know: `hook.Add`, `timer.Create`, `player.GetAll`, `ply:Nick()`, `DarkRP.createJob`…

Addons are plain folders of `.lua` files dropped on your server, **no compilation, no republishing**:

```text
data/rebornstudio/cityliferp/addons/
└── my-addon/
    ├── addon.json      (manifest, optional)
    └── main.lua        (entry point)
```

```lua
-- main.lua, a first addon in 6 lines
hook.Add("PlayerInitialSpawn", "hello", function(ply)
	timer.Simple(8, function()
		if IsValid(ply) then ply:ChatPrint("Welcome to my server!") end
	end)
end)
```

Hot-reload with `addon_reload` in the server console. A failing addon is isolated and logged, **it never brings the server down**.

## The philosophy

- **Your addons, your server.** The SDK enforces no gameplay rules: build the jobs and systems YOU want.
- **Security first.** Every addon runs in an isolated sandbox (no disk access outside its folder, per-call CPU budget, curated API).
- **Compatibility measured, not promised.** Every API is validated against real public addons, [the full matrix is public](/en/compat/darkrp).
