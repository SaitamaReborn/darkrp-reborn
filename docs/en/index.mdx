## Build addons — and sell them if you want

The addon system is built so **anyone can extend the game without ever accessing our source code or our databases**: you code against a public API contract, isolated by design. Your addon stays yours — credited in game, and sellable if that is your model.

Your first C# addon, in five minutes:

```csharp
public sealed class MyAddon : RebornAddon
{
	public override string Ident => "my-addon";
	public override string Name => "My first addon";

	public override void OnAddonLoaded()
	{
		RebornEvents.PlayerJoined += p => p.Chat( $"Hello {p.Name}!" );
		Reborn.RegisterChatCommand( "hi", "Says hi", ( player, args ) =>
			player.Notify( "Hi!", $"You are carrying {player.Money}$." ), Ident );
	}
}
```

No registration: every class deriving from `RebornAddon` is discovered and loaded automatically, server side. The same code runs in the Dev Kit simulator and in production. [Start now →](/en/addons/csharp-getting-started)

## The philosophy

- **Your addons, your server.** No gameplay rules imposed: build the jobs and systems YOU want, keep ownership of your code.
- **Security first.** Every addon runs isolated: no database access, sandboxed persistence, per-call CPU budget on the Lua side, server-authoritative economy on the C# side.
- **Measured compatibility, not promised.** Every GMod API is validated against real public addons — [the full matrix is public](/en/compat/darkrp).
