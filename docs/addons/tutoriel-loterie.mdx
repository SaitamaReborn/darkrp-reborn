On construit une **loterie municipale** : les joueurs achètent des tickets à 100 €, un tirage automatique a lieu toutes les 20 minutes, le gagnant rafle une cagnotte qui survit aux redémarrages. Le code complet est dans `examples/loterie/` du SDK.

├── addon.json
└── main.lua
```

```

## 2. Configuration et traductions

```lua
local TICKET_PRICE = 100
local DRAW_MINUTES = 20
local POT_SEED     = 500

locale.Register("loterie.ticket_ok",
	"Ticket acheté ! Vous en avez %d. Cagnotte : %s €.",
	"Ticket bought! You now hold %d. Pot: %s €.")
-- (les autres clés sur le même modèle)
```

Règle d'or : **aucun texte en dur**, tout passe par `locale.Register` en fr + en.

## 3. L'état persistant

```lua
local tickets = data.Get("tickets", {})   -- { [steamid64] = nombre }
local pot     = data.Get("pot", POT_SEED)

local function save()
	data.Set("tickets", tickets)
	data.Set("pot", pot)
end
```

`data.Get(clé, défaut)` rend le défaut au premier lancement. Les SteamID64 servent de clés :
**toujours en string** (précision des nombres Lua oblige).

## 4. Acheter un ticket

```lua
chat.AddCommand("loterie", function(ply, args, raw)
	if !ply:canAfford(TICKET_PRICE) then
		ply:Notify("danger", "Loterie", string.format(locale.T("loterie.pas_assez"), TICKET_PRICE))
		return
	end
	ply:takeMoney(TICKET_PRICE, "Ticket de loterie")

	local sid = ply:SteamID64()
	tickets[sid] = (tickets[sid] or 0) + 1
	pot = pot + TICKET_PRICE
	save()

	ply:Notify("success", "Loterie",
		string.format(locale.T("loterie.ticket_ok"), tickets[sid], string.Comma(pot)))
end, "Acheter un ticket de loterie")
```

À noter : la syntaxe GLua (`!ply:canAfford(...)`) et `takeMoney` qui gère lui-même l'affichage du débit.

## 5. Le tirage automatique

```lua
timer.Create("loterie_draw", DRAW_MINUTES * 60, 0, function()
	local pool = {}
	for sid, n in pairs(tickets) do
		for i = 1, n do pool[#pool + 1] = sid end   -- 1 entrée par ticket
	end
	if #pool == 0 then return end                    -- personne : la cagnotte roule

	local winSid = pool[math.random(#pool)]
	local gain = pot
	tickets, pot = {}, POT_SEED                      -- nouvelle manche
	save()

	local winner = player.GetBySteamID64(winSid)
	if IsValid(winner) then
		winner:addMoney(gain, "Gain de loterie")
		chat.Broadcast("🎰 " .. string.format(locale.T("loterie.gagnant"), winner:Nick(), string.Comma(gain)))
	else
		chat.Broadcast("🎰 " .. locale.T("loterie.gagnant_absent"))
	end
end)
```

`timer.Create(id, délai, 0, fn)` : répétitions `0` = infini. Le gagnant peut être déconnecté au moment du tirage, d'où le `IsValid`.

## 6. L'accueil des joueurs

```lua
hook.Add("PlayerInitialSpawn", "loterie_bienvenue", function(ply)
	timer.Simple(15, function()
		if IsValid(ply) then
			ply:Notify("info", "Loterie", string.format(locale.T("loterie.bienvenue"), string.Comma(pot)))
		end
	end)
end)
```

Le délai laisse l'écran de sélection de personnage se terminer avant la notification.

## 7. Déployer et itérer

```text
1. SFTP → data/rebornstudio/cityliferp/addons/loterie/
2. Console : addon_reload loterie
3. addon_list → « Loterie de la Ville » v1.0 : Loaded
4. En jeu : /loterie, /loterie_info
```

Chaque `addon_reload` repart des fichiers du disque et ré-attache proprement commandes, timer et hook. La cagnotte, elle, reste en place, c'est le rôle de `data.*`.
