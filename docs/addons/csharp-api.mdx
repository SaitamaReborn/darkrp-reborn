Tout le contrat tient dans trois fichiers, livrés à l'identique dans le jeu et
dans le Dev Kit (`devkit/code/Addons/`). Votre addon ne référence **que** ces
types : jamais les classes internes du gamemode.

## `RebornAddon` : la classe de base

```csharp
public sealed class MonAddon : RebornAddon
{
	public override string Ident => "mon-addon";     // définitif, minuscules-tirets
	public override string Name => "Mon addon";
	public override string Author => "Vous";          // optionnel
	public override string Version => "1.0";          // optionnel

	public override void OnAddonLoaded() { }          // brancher events + commandes ici
	public override void OnAddonUnloaded() { }        // arrêt du serveur
}
```

Un addon **est un `Component` s&box** : `OnStart`, `OnUpdate`, `OnFixedUpdate`
fonctionnent normalement après `OnAddonLoaded`. Chaque addon vit sur son propre
GameObject (`addon:<classe>`, visible dans l'arbre de scène de l'éditeur), côté
**serveur uniquement** — il n'existe pas sur les machines des joueurs.

## `RebornEvents` : les événements du jeu

Mêmes points d'accroche que les hooks Lua/GMod, en typé :

```csharp
RebornEvents.ServerReady   += () => { };                          // boot terminé
RebornEvents.PlayerJoined  += ( IRebornPlayer p ) => { };         // première apparition
RebornEvents.PlayerSpawned += ( IRebornPlayer p ) => { };         // chaque (re)spawn
RebornEvents.PlayerLeft    += ( IRebornPlayer p ) => { };         // déconnexion
RebornEvents.PlayerDied    += ( IRebornPlayer p ) => { };         // mort
RebornEvents.PlayerChat    += ( RebornChatEvent e ) => { };       // e.Speaker, e.Text (observation)
RebornEvents.JobChanged    += ( p, ancienMetier, nouveauMetier ) => { };
```

Un handler qui lève une exception est isolé : il n'éteint ni les autres addons
ni le serveur (l'erreur part dans le log serveur).

## `Reborn` : l'API statique

```csharp
// Joueurs
IEnumerable<IRebornPlayer> tous = Reborn.Players;
IRebornPlayer p = Reborn.FindPlayer( "Alice" );        // nom (préfixe) ou SteamID64

// Chat & notifications
Reborn.Broadcast( "Annonce à tout le serveur" );
Reborn.BroadcastNotify( "Titre", "Message", RebornNotifyType.Info );

// Commandes chat (/macommande) — le natif puis les addons Lua ont priorité :
// un addon C# ne peut jamais masquer une commande du jeu.
Reborn.RegisterChatCommand( "macommande", "Description", ( player, args ) => { }, Ident );

// Timers (côté serveur, pompés par le chargeur)
Reborn.After( 10f, () => { } );                        // une fois, dans 10 s
Reborn.Every( 300f, () => { } );                       // toutes les 5 min

// Persistance : un JSON par addon, survit aux restarts
Reborn.SaveData( Ident, json );
string json = Reborn.LoadData( Ident );                // null si absent

// Log serveur préfixé [addon:ident]
Reborn.Log( Ident, "message" );

// Monde : spawn d'un prop physique (serveur réel ; null dans le Dev Kit)
var crate = Reborn.SpawnProp( "models/citizen_props/crate01.vmdl", pos, 45f, Ident );

// Contenu runtime : métiers dans le F4 et items dans l'inventaire, comme les natifs
Reborn.RegisterJob( new RebornJobSpec { Name = "Livreur de pizzas", Salary = 320f, MaxWorkers = 4 } );
Reborn.RegisterItem( new RebornItemSpec { Id = "pizza", Name = "Pizza", Value = 25, HungerRestore = 40f } );
```

## `IRebornPlayer` : le joueur

```csharp
p.Name          // pseudo (Steam sur le vrai serveur)
p.SteamId       // SteamID64 en string
p.Job           // nom du métier ("Policier", ...)
p.Money         // argent liquide
p.Health        // vie (0..100)
p.Position      // position monde (Vector3.Zero dans le Dev Kit)
p.Language      // langue de son interface ("fr", "en", ...)
p.GameObject    // pawn réel sur le serveur ; null dans le Dev Kit

p.GiveMoney( 100, "Récompense" );          // autoritaire côté serveur
bool ok = p.TakeMoney( 50, "Amende" );     // false si pas les moyens (rien pris)
p.SetMoney( 5000 );                        // fixe le liquide à un montant exact
p.Teleport( new Vector3( 100, 200, 64 ) ); // téléporte le pawn (côté serveur)
p.SetHealth( 50f );                        // 0..100, 0 = létal
p.TrySetJob( "Policier" );                 // id ou nom ; règles normales du jeu
p.GiveItem( "pizza", 3 );                  // items catalogue OU enregistrés par addon
p.Notify( "Titre", "Message", RebornNotifyType.Success );
p.Chat( "Ligne privée dans son chat" );
```

`GiveMoney`/`TakeMoney` passent par le même circuit anti-triche que le reste de
l'économie (gate host + garde-fous). Aucun addon ne peut créer d'argent côté
client.

## Règles et limites (v2)

- **Serveur uniquement** : pas d'UI côté client ni de RPC custom. Pour un
  retour visuel, utilisez `Notify`/`Chat`. Le détail de ce qui reste fermé (et
  pourquoi) est sur [la page limites](/addons/csharp-limites).
- `p.GameObject` est le pawn réel : positions, spawn de props à proximité...
  Testez toujours `!= null` (Dev Kit) et gardez les mutations côté serveur.
- Ne touchez pas aux fichiers du contrat (`code/Addons/`) ni aux types internes
  du jeu : c'est le critère n°1 de la revue d'intégration.
- Textes joueurs : `p.Language` vous donne la langue du joueur si vous voulez
  localiser vos messages (10 langues côté jeu).
